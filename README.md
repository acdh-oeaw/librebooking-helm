# LibreBooking Helm Chart

Helm chart to deploy LibreBooking (apache + PHP application) to Kubernetes using official images and the documented environment variables.

This chart supports:
- LibreBooking app container (official image)
- Internal MariaDB (official `mariadb` image) or an external database via `LB_DATABASE_HOSTSPEC`
- Required LB_* environment variables from RUN.md
- Optional persistence for logs and uploads
- Optional background jobs deployment using `supercronic /config/lb-jobs-cron`
- Optional favicon override via Secret/ConfigMap
- Optional Ingress

Links
- LibreBooking Docker repo: https://github.com/LibreBooking/docker
- RUN.md: https://github.com/LibreBooking/docker/blob/master/RUN.md
- SETUP.md: https://github.com/LibreBooking/docker/blob/master/SETUP.md

## Requirements

- Kubernetes 1.23+ (Ingress v1 API used)
- Helm 3.x
- Storage class available if you enable persistence
- A MariaDB or MySQL endpoint if using external DB
- For initial setup, an installation password (LB_INSTALL_PASSWORD) in a Secret

## Quick start

1) Install with internal MariaDB (recommended for first-time setup):
```
# Edit values.yaml to set:
# - mariadb.auth.rootPassword (required for installer)
# - app.installPassword (or app.existingSecretName)
helm install my-librebooking librebooking/
```

2) Or install with an external DB:
```
# Edit librebooking/external-db-values.yaml to set hostspec, db name/user/password, and install password
helm install my-librebooking librebooking/ -f librebooking/external-db-values.yaml
```

3) Access the app:
- If Ingress enabled: https://<your-host>[/<APP_PATH>]
- Without Ingress (port-forward):
  ```
  kubectl port-forward svc/my-librebooking-librebooking 8080:80
  open http://127.0.0.1:8080/
  ```

## First-time setup (per SETUP.md)

- Open http(s)://<HOST>[/<APP_PATH>]/install
- Enter:
  - Installation password: LB_INSTALL_PASSWORD
  - DB root user: root
  - DB root password:
    - Internal DB: from Secret `<release>-librebooking-mariadb` key `mysql-root-password`
    - External DB: your DB’s root password
- Choose “Create the database” and “Create the database user” if needed
- Complete registration of the administrator and configure the app

## Configuration

This chart maps the official variables described in RUN.md:

Required by the app:
- LB_DATABASE_NAME
- LB_DATABASE_USER
- LB_DATABASE_PASSWORD
- LB_DATABASE_HOSTSPEC
- LB_INSTALL_PASSWORD
- LB_DEFAULT_TIMEZONE
- LB_LOGGING_FOLDER (default /var/log/librebooking)
- LB_LOGGING_LEVEL (default none)
- LB_LOGGING_SQL (default false)

Optional:
- APP_PATH

The chart exposes these as Helm values.

### Key values in values.yaml

Application
- app.defaultTimezone: LB_DEFAULT_TIMEZONE (default: UTC)
- app.installPassword: Used to create a Secret for LB_INSTALL_PASSWORD when app.existingSecretName is empty
- app.existingSecretName / app.existingSecretKey: Existing Secret containing LB_INSTALL_PASSWORD
- app.loggingFolder: LB_LOGGING_FOLDER (default: /var/log/librebooking)
- app.loggingLevel: LB_LOGGING_LEVEL (default: none)
- app.loggingSql: LB_LOGGING_SQL (default: "false")
- app.appPath: APP_PATH (e.g., /librebooking)

Database
- externalDatabase.enabled: set true to use external DB
- externalDatabase.hostspec: LB_DATABASE_HOSTSPEC, e.g. "db.example.com:3306"
- externalDatabase.name/user/password: LB_DATABASE_NAME/USER/PASSWORD (password can be sourced from existingSecretName/existingSecretKey)
- mariadb.enabled: set true to deploy internal MariaDB
- mariadb.auth.rootPassword: required for web installer
- mariadb.auth.database/username/password: pre-creates DB and user in the MariaDB container

Persistence (PVCs)
- persistence.logs.enabled + size/storageClass/existingClaim
  - Mounted at LB_LOGGING_FOLDER (default /var/log/librebooking)
- persistence.uploadsImages.enabled + size/storageClass/existingClaim
  - Mounted at /var/www/html/Web/uploads/images
- persistence.uploadsReservation.enabled + size/storageClass/existingClaim
  - Mounted at /var/www/html/Web/uploads/reservation

Favicon
- favicon.enabled: true to mount a single file to /var/www/html/Web/favicon.ico
- favicon.type: secret or configMap
- favicon.name/key: the name/key where the icon is stored

Background jobs (optional)
- jobs.enabled: Deploys a “jobs” Deployment that runs `supercronic /config/lb-jobs-cron`
- jobs.image.*: Image settings for the jobs pod (defaults to same LibreBooking image)

Service/Ingress
- service.type: ClusterIP, NodePort, or LoadBalancer (default: ClusterIP)
- ingress.enabled + className/hosts/tls: Enable and configure Ingress

Resources
- resources: CPU/memory requests/limits for the app container
- mariadb.primary.resources: resources for the internal DB

### Example: external DB values

librebooking-helm/external-db-values.yaml (provided):

```
mariadb:
  enabled: false

externalDatabase:
  enabled: true
  hostspec: "my-db.example.com:3306"
  name: "librebooking"
  user: "librebooking"
  password: "CHANGE_ME"
  # Or use an existing secret:
  # existingSecretName: my-db-secret
  # existingSecretKey: db-password

app:
  defaultTimezone: "UTC"
  installPassword: "CHANGE_ME"

ingress:
  enabled: true
  className: "nginx"
  hosts:
    - host: librebooking.example.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - secretName: librebooking-tls
      hosts:
        - librebooking.example.com
```

## Security notes

- Do not commit real secrets to Git.
- Prefer Kubernetes Secrets (`app.existingSecretName`, `externalDatabase.existingSecretName`, `mariadb.auth.existingSecretName`) over inline plaintext in values.
- Set a strong `mariadb.auth.rootPassword` for the internal database during setup.
- Restrict access to the service with NetworkPolicies if needed.

## Upgrades

- To upgrade the chart:

- Application upgrades may require visiting `/install/configure.php` as per SETUP.md.

## Troubleshooting

- Pod not ready (app):
  - Check logs: `kubectl logs deploy/my-librebooking-librebooking`
  - Verify DB connectivity: `LB_DATABASE_HOSTSPEC`, name/user/password
  - Ensure `LB_INSTALL_PASSWORD` Secret is present and referenced correctly

- Internal DB connection issues:
  - Ensure mariadb pod is running: `kubectl get pods`
  - Confirm Secret keys: `mysql-root-password` and `mysql-password`
  - Service name should resolve to: `<release>-librebooking-mariadb:3306`

- Installer page doesn’t load:
  - Port-forward to the service and open `/install`
  - If using `APP_PATH`, access `/<APP_PATH>/install`
  - Verify Ingress path and rewrite behavior match `APP_PATH`

- File permissions for persistence:
  - Ensure the container user can write to mounted PVC paths
  - Some storage classes may need supplemental fsGroup or securityContext adjustments

- Background jobs not running:
  - Set `jobs.enabled=true`
  - Check logs: `kubectl logs deploy/my-librebooking-librebooking-jobs`
